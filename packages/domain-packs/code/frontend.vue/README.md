# Vue Frontend Engineering

Pack id: code.frontend.vue

## Boundary
Covers Vue single-file components, Composition API, template reactivity, Pinia, Vue Router, Nuxt/SSR hydration, Vite integration, and browser smoke validation.

## Not Covered
Does not duplicate React hook semantics or generic Web platform/a11y/CSS rules; combine with code.frontend.web for browser fundamentals and accessibility.

## Dependencies
code.core, code.javascript, code.frontend.web

## Evidence Rules
Positive documents use medium or strong evidence only. Skills may activate at high/max/ultra; strategy, methodology, and knowledge entries are limited to max/ultra. Failure dossiers remain under documents/failure_dossiers and are not indexed as positive refs.

## L3 Validation

Representative activation and retrieval smoke lives in `evals/smoke/l3-smoke.json`. The quality report lives in `quality/l3-report.json`. A pack reaches L3 only when activation, pack-scoped retrieval, validation smoke, diagnosis smoke, and failure dossier boundaries are all present.
