# Code Core

## Boundary

Covers core engineering behavior tasks where repository evidence touches repository evidence, git state, user instructions, and focused validation output.

## Out of Scope

Does not override more specific domain packs, project instructions, runtime permissions, or human approval gates. Use this pack as bounded engineering guidance, not as a broad product or policy authority.

## Applies When

Use this pack when detector evidence, repository files, or the user request clearly match code.

## Evidence Rules

Positive documents use medium or strong evidence only. Skills may activate at high/max/ultra; strategy, methodology, knowledge, and memory entries are limited to max/ultra. Failure dossiers remain under documents/failure_dossiers and are not indexed as positive refs.

## Content Shape

Strategies: 6; methodologies: 4; knowledge: 5; skills: 3; failure dossiers: 4.

## L3 Validation

Representative activation and retrieval smoke lives in `evals/smoke/l3-smoke.json`. The quality report lives in `quality/l3-report.json`. A pack reaches L3 only when activation, pack-scoped retrieval, validation smoke, diagnosis smoke, and failure dossier boundaries are all present.
