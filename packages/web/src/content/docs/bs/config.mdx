## Format


## Lokacije

MoŇĺete postaviti svoju konfiguraciju na nekoliko razlińćitih lokacija i one imaju drugańćiji redoslijed prioriteta.

:::note
Konfiguracijski fajlovi se **spajaju**, ne zamjenjuju.
:::

Konfiguracijski fajlovi se spajaju, ne zamjenjuju. Kombiniraju se postavke sa sljedeńáih konfiguracijskih lokacija. Kasnije konfiguracije poniŇ°tavaju prethodne samo za konfliktne kljuńćeve. Nekonfliktne postavke iz svih konfiguracija su sańćuvane.

Na primjer, ako vaŇ°a globalna konfiguracija postavlja `autoupdate: true`, a vaŇ°a projektna konfiguracija postavlja `model: "anthropic/claude-sonnet-4-5"`, konańćna konfiguracija ńáe ukljuńćivati ‚Äč‚Äčobje postavke.

---

### Redoslijed prioriteta

Izvori konfiguracije se uńćitavaju ovim redoslijedom (kasniji izvori poniŇ°tavaju ranije):

1. **Udaljena konfiguracija** (od `.well-known/deepagent-code`) - organizacijske postavke
2. **Globalna konfiguracija** (`~/.config/deepagent-code/deepagent-code.json`) - korisnińćke preferencije
3. **PrilagońĎena konfiguracija** (`DEEPAGENT_CODE_CONFIG` env var) - prilagońĎena preinańćenja
4. **Konfiguracija projekta** (`deepagent-code.json` u projektu) - postavke specifińćne za projekat
5. **`.deepagent-code` direktoriji** - agenti, komande, dodaci
6. **Inline konfiguracija** (`DEEPAGENT_CODE_CONFIG_CONTENT` env var) - runtime preinańćenja

To znańći da konfiguracije projekta mogu nadjańćati globalne zadane postavke, a globalne konfiguracije mogu nadjańćati postavke udaljene organizacije.

:::note
Direktoriji `.deepagent-code` i `~/.config/deepagent-code` koriste **imena u mnoŇĺini** za poddirektorije: `agents/`, `commands/`, `modes/`, `plugins/`, `skills/`, `tools/` i `themes/`. Pojedinańćna imena (npr. `agent/`) su takońĎer podrŇĺana za kompatibilnost unatrag.
:::

### Udaljeno (Remote)

Organizacije mogu pruŇĺiti zadanu konfiguraciju preko `.well-known/deepagent-code` krajnje tańćke. Ovo se automatski preuzima kada se autentifikujete kod provajdera koji to podrŇĺava.

Prvo se uńćitava udaljena konfiguracija koja sluŇĺi kao osnovni sloj. Svi ostali izvori konfiguracije (globalni, projektni) mogu nadjańćati ove zadane postavke.

Na primjer, ako vaŇ°a organizacija nudi MCP servere koji su po defaultu onemoguńáeni:

```json title="Remote config from .well-known/deepagent-code"
{
  "mcp": {
    "jira": {
      "type": "remote",
      "url": "https://jira.example.com/mcp",
      "enabled": false
    }
  }
}
```

MoŇĺete omoguńáiti odreńĎene servere u vaŇ°oj lokalnoj konfiguraciji:

```json title="deepagent-code.json"
{
  "mcp": {
    "jira": {
      "type": "remote",
      "url": "https://jira.example.com/mcp",
      "enabled": true
    }
  }
}
```

### Globalno

Postavite svoju globalnu DeepAgent Code konfiguraciju u `~/.config/deepagent-code/deepagent-code.json`. Koristite globalnu konfiguraciju za korisnińćke preferencije kao Ň°to su provajderi, modeli i dozvole.

Za postavke specifińćne za TUI, koristite `~/.config/deepagent-code/tui.json`.

Globalna konfiguracija poniŇ°tava zadane postavke udaljene organizacije.

---

### Projekt

Dodajte `deepagent-code.json` u korijen projekta. Konfiguracija projekta ima najveńái prioritet meńĎu standardnim konfiguracijskim datotekama - ona nadjańćava globalne i udaljene konfiguracije.

Za TUI postavke specifińćne za projekat, dodajte `tui.json` pored njega.

:::tip
Postavite specifińćnu konfiguraciju projekta u korijen vaŇ°eg projekta.
:::

Kada se DeepAgent Code pokrene, traŇĺi konfiguracijsku datoteku u trenutnom direktoriju ili prelazi do najbliŇĺeg Git direktorija.

Ovo je takońĎer sigurno provjeriti u Git i koristi istu shemu kao globalna.

---

### PrilagońĎena konfiguracija

Navedite prilagońĎenu putanju konfiguracijske datoteke koristeńái varijablu okruŇĺenja `DEEPAGENT_CODE_CONFIG`.

```bash

deepagent-code run "Hello world"
```

PrilagońĎena konfiguracija se uńćitava izmeńĎu globalne i projektne konfiguracije po redoslijedu prioriteta.

---

### PrilagońĎeni direktorij

Navedite prilagońĎeni konfiguracijski direktorij koristeńái `DEEPAGENT_CODE_CONFIG_DIR` varijablu okruŇĺenja. U ovom direktoriju ńáe se traŇĺiti agenti, komande, modovi i dodaci baŇ° kao standardni `.deepagent-code` direktorij, i trebali bi pratiti istu strukturu.

```bash

deepagent-code run "Hello world"
```

PrilagońĎeni direktorij se uńćitava nakon direktorija globalne konfiguracije i `.deepagent-code`, tako da **moŇĺe nadjańćati** njihove postavke.

### Server

MoŇĺete konfigurirati postavke servera za naredbe `deepagent-code serve` i `deepagent-code web` putem opcije `server`.

```json title="deepagent-code.json"
{
  "$schema": "https://deepagent-code.ai/config.json",
  "server": {
    "port": 4096,
    "hostname": "0.0.0.0",
    "mdns": true,
    "mdnsDomain": "myproject.local",
    "cors": ["http://localhost:5173"]
  }
}
```

Dostupne opcije:

- `port` - Port za sluŇ°anje.
- `hostname` - Hostname za sluŇ°anje. Kada je `mdns` omoguńáen i nije postavljeno ime hosta, podrazumevano je `0.0.0.0`.
- `mdns` - Omoguńái mDNS otkrivanje servisa. Ovo omoguńáava drugim ureńĎajima na mreŇĺi da otkriju vaŇ° DeepAgent Code server.
- `mdnsDomain` - PrilagońĎeno ime domene za mDNS servis. Zadano je `deepagent-code.local`. Korisno za pokretanje viŇ°e instanci na istoj mreŇĺi.
- `cors` - Dodatni origini koji omoguńáavaju CORS kada koristite HTTP server iz klijenta baziranog na pretraŇĺivańću. Vrijednosti moraju biti puni origin (shema + host + opcijski port), npr. `https://app.example.com`.

[Saznajte viŇ°e o serveru ovdje](/docs/server).

### Model

MoŇĺete konfigurirati dobavljańće i modele koje Ňĺelite koristiti u svojoj DeepAgent Code konfiguraciji kroz opcije `provider`, `model` i `small_model`.

```json title="deepagent-code.json"
{
  "$schema": "https://deepagent-code.ai/config.json",
  "provider": {},
  "model": "anthropic/claude-sonnet-4-5",
  "small_model": "anthropic/claude-haiku-4-5"
}
```

Opcija `small_model` konfigurira poseban model za lagane zadatke poput generiranja naslova. Podrazumevano, DeepAgent Code pokuŇ°ava da koristi jeftiniji model ako je dostupan od vaŇ°eg provajdera, inańće se vrańáa na vaŇ° glavni model.

Opcije provajdera mogu ukljuńćivati ‚Äč‚Äč`timeout` i `setCacheKey`:

```json title="deepagent-code.json"
{
  "$schema": "https://deepagent-code.ai/config.json",
  "provider": {
    "anthropic": {
      "options": {
        "timeout": 600000,
        "setCacheKey": true
      }
    }
  }
}
```

- `timeout` - Vrijeme ńćekanja zahtjeva u milisekundama (podrazumevano: 300000). Postavite na `false` da onemoguńáite.
- `setCacheKey` - Osigurajte da je kljuńć keŇ° memorije uvijek postavljen za odreńĎenog provajdera.

TakońĎer moŇĺete konfigurirati [lokalne modele](/docs/models#local). [Saznajte viŇ°e](/docs/models).

#### Opcije specifińćne za provajdere

Neki provajderi podrŇĺavaju dodatne opcije konfiguracije osim generińćkih postavki `timeout` i `apiKey`.

##### Amazon Bedrock

Amazon Bedrock podrŇĺava konfiguraciju specifińćnu za AWS:

```json title="deepagent-code.json"
{
  "$schema": "https://deepagent-code.ai/config.json",
  "provider": {
    "amazon-bedrock": {
      "options": {
        "region": "us-east-1",
        "profile": "my-aws-profile",
        "endpoint": "https://bedrock-runtime.us-east-1.vpce-xxxxx.amazonaws.com"
      }
    }
  }
}
```

- `region` - AWS regija za Bedrock (zadano na `AWS_REGION` env var ili `us-east-1`)
- `profile` - AWS imenovani profil iz `~/.aws/credentials` (zadano na `AWS_PROFILE` env var)
- `endpoint` - URL prilagońĎene krajnje tańćke za VPC krajnje tańćke. Ovo je alias za generińćku opciju `baseURL` koristeńái terminologiju specifińćnu za AWS. Ako su oba navedena, `endpoint` ima prednost.

:::note
Tokeni nosioca (`AWS_BEARER_TOKEN_BEDROCK` ili `/connect`) imaju prednost nad autentifikacijom zasnovanom na profilu. Pogledajte [prednost autentifikacije](/docs/providers#authentication-precedence) za detalje.
:::

[Saznajte viŇ°e o konfiguraciji Amazon Bedrock](/docs/providers#amazon-bedrock).

### Zadani agent

MoŇĺete postaviti zadanog agenta koristeńái opciju `default_agent`. Ovo odreńĎuje koji se agent koristi kada nijedan nije eksplicitno specificiran.

Zadani agent mora biti primarni agent (ne podagent). Ovo moŇĺe biti ugrańĎeni agent kao Ň°to je `"build"` ili `"plan"`, ili [prilagońĎeni agent](/docs/agents) koji ste definirali. Ako navedeni agent ne postoji ili je podagent, DeepAgent Code ńáe se vratiti na `"build"` s upozorenjem.

### Onemoguńáeni provajderi

MoŇĺete onemoguńáiti dobavljańće koji se automatski uńćitavaju preko opcije `disabled_providers`. Ovo je korisno kada Ňĺelite sprijeńćiti uńćitavanje odreńĎenih provajdera ńćak i ako su njihovi vjerodajnici dostupni.

```json title="deepagent-code.json"
{
  "$schema": "https://deepagent-code.ai/config.json",
  "disabled_providers": ["openai", "gemini"]
}
```

:::note
`disabled_providers` ima prioritet nad `enabled_providers`.
:::

Opcija `disabled_providers` prihvata niz ID-ova provajdera. Kada je provajder onemoguńáen:

- Neńáe se uńćitati ńćak i ako su varijable okruŇĺenja postavljene.
- Neńáe se uńćitati ńćak i ako su API kljuńćevi konfigurirani putem `/connect` naredbe.
- Modeli dobavljańća se neńáe pojaviti na listi za odabir modela.

### Omoguńáeni provajderi

MoŇĺete odrediti listu dozvoljenih dobavljańća putem opcije `enabled_providers`. Kada se podesi, samo navedeni provajderi ńáe biti omoguńáeni, a svi ostali ńáe biti zanemareni.

```json title="deepagent-code.json"
{
  "$schema": "https://deepagent-code.ai/config.json",
  "enabled_providers": ["anthropic", "openai"]
}
```

Ovo je korisno kada Ňĺelite da ogranińćite DeepAgent Code da koristi samo odreńĎene provajdere umjesto da ih onemoguńáavate jednog po jednog.

:::note
`disabled_providers` ima prioritet nad `enabled_providers`.
:::

Ako se provajder pojavljuje i u `enabled_providers` i `disabled_providers`, `disabled_providers` ima prioritet za kompatibilnost unatrag.

### Datoteke

Koristite `{file:path/to/file}` da zamijenite sadrŇĺaj fajla:

```json title="deepagent-code.json"
{
  "$schema": "https://deepagent-code.ai/config.json",
  "instructions": ["./custom-instructions.md"],
  "provider": {
    "openai": {
      "options": {
        "apiKey": "{file:~/.secrets/openai-key}"
      }
    }
  }
}
```

Putanja fajla mogu biti:

- U odnosu na direktorij konfiguracijskih datoteka
- Ili apsolutne staze koje pońćinju sa `/` ili `~`

Ovo je korisno za:

- Pohranjivanje osjetljivih podataka poput API kljuńćeva u odvojenim datotekama.
- Ukljuńćujuńái velike datoteke instrukcija bez zatrpavanja vaŇ°e konfiguracije.
- Dijeljenje zajednińćkih isjeńćaka konfiguracije u viŇ°e konfiguracijskih datoteka.
