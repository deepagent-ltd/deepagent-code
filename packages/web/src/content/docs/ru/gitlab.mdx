deepagent-code интегрируется с вашим рабочим процессом GitLab через конвейер GitLab CI/CD или с GitLab Duo.

В обоих случаях deepagent-code будет работать на ваших программах GitLab.

---

## GitLab

deepagent-code работает в обычном конвейере GitLab. Вы можете встроить его в конвейер как [CI-компонент](https://docs.gitlab.com/ee/ci/components/)

Здесь мы используем созданный сообществом компонент CI/CD для deepagent-code — [nagyv/gitlab-deepagent-code](https://gitlab.com/nagyv/gitlab-deepagent-code).

---

### Функции

- **Использовать пользовательскую конфигурацию для каждого задания**. Настройте deepagent-code с помощью пользовательского каталога конфигурации, например `./config/#custom-directory`, чтобы включать или отключать функциональность для каждого вызова deepagent-code.
- **Минимальная настройка**: компонент CI настраивает deepagent-code в фоновом режиме, вам нужно только создать конфигурацию deepagent-code и начальное приглашение.
- **Гибкость**: компонент CI поддерживает несколько входных данных для настройки его поведения.

---

### Настройка

1. Сохраните JSON аутентификации deepagent-code как переменные среды CI типа файла в разделе **Настройки** > **CI/CD** > **Переменные**. Обязательно пометьте их как «Замаскированные и скрытые».
2. Добавьте следующее в файл `.gitlab-ci.yml`.

   ```yaml title=".gitlab-ci.yml"
   include:
     - component: $CI_SERVER_FQDN/nagyv/gitlab-deepagent-code/deepagent-code@2
       inputs:
         config_dir: ${CI_PROJECT_DIR}/deepagent-code-config
         auth_json: $DEEPAGENT_CODE_AUTH_JSON # The variable name for your DeepAgent Code authentication JSON
         command: optional-custom-command
         message: "Your prompt here"
   ```

Дополнительные сведения и варианты использования см. в документации ](https://gitlab.com/explore/catalog/nagyv/gitlab-deepagent-code) для этого компонента.

## GitLab Duo

deepagent-code интегрируется с вашим рабочим процессом GitLab.
Упомяните `@deepagent-code` в комментарии, и deepagent-code выполнит задачи в вашем конвейере GitLab CI.

---

### Функции

- **Триаж задач (Issue Triage)**. Попросите deepagent-code разобраться в проблеме и объяснить ее вам.
- **Исправление и реализация**. Попросите deepagent-code исправить проблему или реализовать функцию.
  Он создаст новую ветку и создаст мерж-реквест с изменениями.
- **Безопасность**: deepagent-code работает на ваших программах GitLab.

---

### Настройка

deepagent-code работает в вашем конвейере GitLab CI/CD. Вот что вам понадобится для его настройки:

:::tip
Ознакомьтесь с [**документацией GitLab**](https://docs.gitlab.com/user/duo_agent_platform/agent_assistant/) для получения актуальных инструкций.
:::

1.  Настройте свою среду GitLab
2.  Настройка CI/CD
3.  Получите ключ API поставщика моделей ИИ
4.  Создать учетную запись службы
5.  Настройка переменных CI/CD
6.  Создайте файл конфигурации потока, вот пример:

        <details>

    <summary>Конфигурация потока</summary>

    ```yaml
    image: node:22-slim
    commands:
      - echo "Installing deepagent-code"
      - npm install --global deepagent-code
      - echo "Installing glab"
      - export GITLAB_TOKEN=$GITLAB_TOKEN_DEEPAGENT_CODE
      - apt-get update --quiet && apt-get install --yes curl wget gpg git && rm --recursive --force /var/lib/apt/lists/*
      - curl --silent --show-error --location "https://raw.githubusercontent.com/upciti/wakemeops/main/assets/install_repository" | bash
      - apt-get install --yes glab
      - echo "Configuring glab"
      - echo $GITLAB_HOST
      - echo "Creating DeepAgent Code auth configuration"
      - mkdir --parents ~/.deepagent/code
      - |
        cat > ~/.deepagent/code/auth.json << EOF
        {
          "anthropic": {
            "type": "api",
            "key": "$ANTHROPIC_API_KEY"
          }
        }
        EOF
      - echo "Configuring git"
      - git config --global user.email "deepagent-code@gitlab.com"
      - git config --global user.name "DeepAgent Code"
      - echo "Testing glab"
      - glab issue list
      - echo "Running DeepAgent Code"
      - |
        deepagent-code run "
        You are an AI assistant helping with GitLab operations.

        Context: $AI_FLOW_CONTEXT
        Task: $AI_FLOW_INPUT
        Event: $AI_FLOW_EVENT

        Please execute the requested task using the available GitLab tools.
        Be thorough in your analysis and provide clear explanations.

        <important>
        Please use the glab CLI to access data from GitLab. The glab CLI has already been authenticated. You can run the corresponding commands.

        If you are asked to summarize an MR or issue or asked to provide more information then please post back a note to the MR/Issue so that the user can see it.
        You don't need to commit or push up changes, those will be done automatically based on the file changes you make.
        </important>
        "
      - git checkout --branch $CI_WORKLOAD_REF origin/$CI_WORKLOAD_REF
      - echo "Checking for git changes and pushing if any exist"
      - |
        if ! git diff --quiet || ! git diff --cached --quiet || [ --not --zero "$(git ls-files --others --exclude-standard)" ]; then
          echo "Git changes detected, adding and pushing..."
          git add .
          if git diff --cached --quiet; then
            echo "No staged changes to commit"
          else
            echo "Committing changes to branch: $CI_WORKLOAD_REF"
            git commit --message "Codex changes"
            echo "Pushing changes up to $CI_WORKLOAD_REF"
            git push https://gitlab-ci-token:$GITLAB_TOKEN@$GITLAB_HOST/gl-demo-ultimate-dev-ai-epic-17570/test-java-project.git $CI_WORKLOAD_REF
            echo "Changes successfully pushed"
          fi
        else
          echo "No git changes detected, skipping push"
        fi
    variables:
      - ANTHROPIC_API_KEY
      - GITLAB_TOKEN_DEEPAGENT_CODE
      - GITLAB_HOST
    ```

        </details>

Подробные инструкции можно найти в [GitLab CLI agents docs](https://docs.gitlab.com/user/duo_agent_platform/agent_assistant/).

### Примеры

Вот несколько примеров того, как вы можете использовать deepagent-code в GitLab.

:::tip
Вы можете настроить использование триггерной фразы, отличной от `@deepagent-code`.
:::

- **Объяснение проблемы**

  Добавьте этот комментарий в issue GitLab.

  ```
  @deepagent-code explain this issue
  ```

  deepagent-code прочитает проблему и ответит с четким объяснением.

- **Исправление проблемы**

  В issue GitLab напишите:

  ```
  @deepagent-code fix this
  ```

  deepagent-code создаст новую ветку, внедрит изменения и откроет мерж-реквест с изменениями.

- **Проверка Merge Request**

  Оставьте следующий комментарий к мерж-реквесту GitLab.

  ```
  @deepagent-code review this merge request
  ```

  deepagent-code рассмотрит мерж-реквест и предоставит отзыв.
