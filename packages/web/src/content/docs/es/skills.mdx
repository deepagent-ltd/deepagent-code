Las habilidades del agente permiten a DeepAgent Code descubrir instrucciones reutilizables de su repositorio o directorio de inicio.
Las habilidades se cargan bajo demanda a travﾃｩs de la herramienta nativa `skill`: los agentes ven las habilidades disponibles y pueden cargar el contenido completo cuando sea necesario.

---

## Colocar archivos

Cree una carpeta por nombre de habilidad y coloque un `SKILL.md` dentro de ella.
DeepAgent Code busca estas ubicaciones:

- Configuraciﾃｳn del proyecto: `.deepagent-code/skills/<name>/SKILL.md`
- Configuraciﾃｳn global: `~/.config/deepagent-code/skills/<name>/SKILL.md`
- Compatible con Proyecto Claude: `.claude/skills/<name>/SKILL.md`
- Compatible con Claude global: `~/.claude/skills/<name>/SKILL.md`
- Compatible con agente de proyecto: `.agents/skills/<name>/SKILL.md`
- Compatible con agentes globales: `~/.agents/skills/<name>/SKILL.md`

---

## Entender el descubrimiento

Para las rutas locales del proyecto, DeepAgent Code sube desde su directorio de trabajo actual hasta llegar al ﾃ｡rbol de trabajo de git.
Carga cualquier `skills/*/SKILL.md` coincidente en `.deepagent-code/` y cualquier `.claude/skills/*/SKILL.md` o `.agents/skills/*/SKILL.md` coincidente a lo largo del camino.

Las definiciones globales tambiﾃｩn se cargan desde `~/.config/deepagent-code/skills/*/SKILL.md`, `~/.claude/skills/*/SKILL.md` y `~/.agents/skills/*/SKILL.md`.

---

## Escribir la introducciﾃｳn

Cada `SKILL.md` debe comenzar con el frontmatter de YAML.
Sﾃｳlo se reconocen estos campos:

- `name` (obligatorio)
- `description` (obligatorio)
- `license` (opcional)
- `compatibility` (opcional)
- `metadata` (opcional, mapa de cadena a cadena)

Los campos desconocidos se ignoran.

---

## Validar nombres

`name` debe:

- Tener entre 1 y 64 caracteres.
- Ser alfanumﾃｩricos en minﾃｺsculas con separadores de guiﾃｳn simple
- No comienza ni termina con `-`
- No contener `--` consecutivos
- Coincide con el nombre del directorio que contiene `SKILL.md`

expresiﾃｳn regular equivalente:

```text
^[a-z0-9]+(-[a-z0-9]+)*$
```

---

## Seguir las reglas de longitud

`description` debe tener entre 1 y 1024 caracteres.
Mantﾃｩngalo lo suficientemente especﾃｭfico para que el agente elija correctamente.

## Usar un ejemplo

Crea `.deepagent-code/skills/git-release/SKILL.md` asﾃｭ:

```markdown
---
name: git-release
description: Create consistent releases and changelogs
license: MIT
compatibility: deepagent-code
metadata:
  audience: maintainers
  workflow: github
---

## What I do

- Draft release notes from merged PRs
- Propose a version bump
- Provide a copy-pasteable `gh release create` command

## When to use me

Ask clarifying questions if the target versioning scheme is unclear.
```

## Reconocer la descripciﾃｳn de la herramienta

DeepAgent Code enumera las habilidades disponibles en la descripciﾃｳn de la herramienta `skill`.
Cada entrada incluye el nombre y la descripciﾃｳn de la habilidad:

```xml
<available_skills>
  <skill>
    <name>git-release</name>
    <description>Create consistent releases and changelogs</description>
  </skill>
</available_skills>
```

El agente carga una habilidad llamando a la herramienta:

```
skill({ name: "git-release" })
```

## Configurar permisos

Controle a quﾃｩ agentes de habilidades pueden acceder utilizando permisos basados 窶銀菊n patrones en `deepagent-code.json`:

```json
{
  "permission": {
    "skill": {
      "*": "allow",
      "pr-review": "allow",
      "internal-*": "deny",
      "experimental-*": "ask"
    }
  }
}
```

| Permiso | Comportamiento                                    |
| ------- | ------------------------------------------------- |
| `allow` | La habilidad se carga inmediatamente              |
| `deny`  | Habilidad oculta al agente, acceso rechazado      |
| `ask`   | Se solicita al usuario aprobaciﾃｳn antes de cargar |

Los patrones admiten comodines: `internal-*` coincide con `internal-docs`, `internal-tools`, etc.

## Anulaciﾃｳn por agente

Otorgue a agentes especﾃｭficos permisos diferentes a los predeterminados globales.

**Para agentes personalizados** (en el frente del agente):

```yaml
---
permission:
  skill:
    "documents-*": "allow"
---
```

**Para agentes integrados** (en `deepagent-code.json`):

```json
{
  "agent": {
    "plan": {
      "permission": {
        "skill": {
          "internal-*": "allow"
        }
      }
    }
  }
}
```

## Deshabilitar la herramienta de habilidades

Deshabilite completamente las habilidades para los agentes que no deberﾃｭan usarlas:

**Para agentes personalizados**:

```yaml
---

  skill: false
---
```

**Para agentes integrados**:

```json
{
  "agent": {
    "plan": {
      "tools": {
        "skill": false
      }
    }
  }
}
```

Cuando estﾃ｡ deshabilitada, la secciﾃｳn `<available_skills>` se omite por completo.

## Solucionar problemas de carga

Si una habilidad no aparece:

1. Verifique que `SKILL.md` estﾃｩ escrito en mayﾃｺsculas.
2. Verifique que el frontmatter incluya `name` y `description`
3. Asegﾃｺrese de que los nombres de las habilidades sean ﾃｺnicos en todas las ubicaciones
4. Verifique los permisos: las habilidades con `deny` estﾃ｡n ocultas para los agentes
